# 敌人物品系统设计文档

## 设计理念

将敌人AI从简单的定时刷怪改为完全镜像玩家的物品系统机制。敌人拥有6格战斗区，使用和玩家完全相同的物品（战士、学徒、野蛮人等），具有相同的CD机制、协同效果和品质系统。

### 核心设计原则
- **完全镜像**：敌人使用和玩家完全相同的物品模板和机制
- **统一规则**：CD计算、协同效果、品质升级完全一致
- **对称体验**：让玩家感受到"和AI在打同一款游戏"

## 系统架构

### 敌人物品系统类
```javascript
class EnemyInventorySystem {
    constructor(game) {
        this.game = game;
        this.inventory = new Array(6).fill(null);  // 6格战斗区
        this.currentWaveConfig = null;
        
        // 直接复用玩家系统的核心组件
        this.itemTemplates = game.inventorySystem.itemTemplates;
        this.qualitySystem = game.inventorySystem.qualitySystem;
    }
    
    // 核心更新循环：完全镜像玩家逻辑
    update() {
        this.updateItemCooldowns();  // CD倒计时
        this.autoProduceUnits();     // 自动生产单位
        this.applyAllSynergyEffects(); // 协同效果
    }
}
```

### 核心机制：完全镜像玩家
1. **CD系统**：使用完全相同的 `getActualCooldown()` 逻辑
2. **协同效果**：野蛮人给相邻近战加攻击，徽章减少CD等
3. **品质系统**：敌人物品也有普通→神话的品质升级
4. **特殊技能**：忍者购买近战自强化、泰坦充能等

## 核心实现机制

### 完全镜像玩家逻辑的实现方法
```javascript
class EnemyInventorySystem {
    // 直接复用玩家的所有核心函数
    getActualCooldown(item) {
        return this.game.inventorySystem.getActualCooldown(item);
    }
    
    applyBarbarianBonus() {
        // 复用玩家的野蛮人逻辑，但作用于enemy.inventory
        return this.game.inventorySystem.applyBarbarianBonus.call(this);
    }
    
    refreshAllBadgeEffects() {
        // 复用玩家的徽章逻辑
        return this.game.inventorySystem.refreshAllBadgeEffects.call(this);
    }
    
    produceUnits(item, itemSlot) {
        // 复用玩家的生产逻辑，但调用enemy版本的单位生成
        // ... 完全相同的加成计算 ...
        this.game.spawnEnemyUnitBySpecificType(item.id, unitType, bonusAttack, bonusHealth, item.quality);
    }
}
```

### 波次装备系统
```javascript
// 每波开始时装备指定配置
initializeWave(waveNumber) {
    const config = enemyWaveConfigs[waveNumber];
    
    // 清空当前装备
    this.inventory = new Array(6).fill(null);
    
    // 按配置装备物品
    config.layout.forEach(({item, slot, quality}) => {
        const newItem = this.createItemInstance(item, quality);
        this.inventory[slot] = newItem;
        
        // 初始化CD（随机0-50%，避免同步刷怪）
        const actualCooldown = this.getActualCooldown(newItem);
        newItem.cooldownRemaining = Math.floor(Math.random() * actualCooldown * 0.5);
    });
    
    // 应用所有协同效果
    this.refreshAllSynergyEffects();
}
```

## 技术优势

### 1. 零学习成本
- 敌人使用玩家完全相同的规则
- 玩家看到敌人配置就知道威胁程度
- 不需要额外的AI逻辑或平衡调整

### 2. 自动平衡
- 敌人强度完全取决于物品品质和配置
- 新增玩家物品自动适用于敌人
- 玩家物品的平衡调整自动影响敌人

### 3. 策略深度
- 玩家需要针对敌人的具体配置制定策略
- 看到"野蛮人+近战"就知道需要准备高血量单位
- 看到"学徒+法师"就知道需要准备抗魔法策略

### 4. 可视化友好
- 可以直接显示敌人的6格装备界面
- 玩家一目了然敌人的战术配置
- 增加战术透明度和策略思考

### 难度递进设计

#### 品质递进
- **第1-3波**：全1级品质（普通），让玩家熟悉基础机制
- **第4-5波**：引入2级品质（优秀），开始协同效果  
- **第6-8波**：2-3级混合，加入大型物品（稀有）
- **第9波**：稳定3级品质，复杂协同组合
- **第10波**：全4级史诗品质，终极挑战

#### 协同复杂度递进
1. **前期（1-3波）**：单纯物品效果，无复杂协同
2. **中期（4-6波）**：引入基础协同（学徒+魔法、野蛮人+近战）
3. **后期（7-9波）**：多重协同（魔法三件套、近战双核心）
4. **终极（10波）**：满配协同，所有机制叠加

#### 物品尺寸递进
- **1-5波**：主要1格物品
- **6-7波**：引入2格物品（巨人、魔偶）
- **8波**：引入3格物品（泰坦）
- **9-10波**：各种尺寸物品混合配置

### 策略对抗指南

玩家看到敌人配置时的应对策略：

#### 看到野蛮人+近战组合
- 准备高血量坦克单位吸收高攻击伤害
- 优先击杀野蛮人，削弱整体攻击力

#### 看到学徒+魔法组合  
- 准备抗魔法单位或高血量单位
- 优先击杀学徒，切断协同加成

#### 看到战旗+全场加速
- 准备快速击杀战旗的高机动性单位
- 预期敌人生产速度会阶段性暴增

#### 看到大型物品（巨人、泰坦）
- 准备群体攻击或高穿透攻击
- 利用数量优势围攻高价值目标
