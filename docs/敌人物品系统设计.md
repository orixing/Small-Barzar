# 敌人物品系统设计文档

## 设计理念

将敌人AI从简单的定时刷怪改为基于物品系统的机制，敌人也拥有一个6格的隐藏战斗区，通过不同的"兵营物品"来生产单位。这样可以实现：
- 玩家与敌人机制统一
- 更灵活的波次配置
- 更有策略性的敌人设计

## 系统架构

### 敌人物品系统类
```javascript
class EnemyInventorySystem {
    constructor(game) {
        this.game = game;
        this.inventory = new Array(6).fill(null);
        this.enemyGold = 0;
        this.currentWaveConfig = null;
    }
}
```

### 核心机制
1. **物品CD系统**：复用现有的冷却机制
2. **触发生产**：CD转好时生产对应兵种
3. **金币消耗**：生产单位时扣除金币
4. **动态配置**：每波切换不同的物品配置

## 实现步骤

### 第一阶段：基础框架
1. 创建 `EnemyInventorySystem` 类
2. 定义基础物品模板
3. 实现物品CD和触发机制
4. 集成到现有游戏循环

### 第二阶段：波次配置
1. 创建波次配置数据
2. 实现动态装备切换
3. 添加波次预览功能
4. 平衡性测试和调整

### 第三阶段：高级功能
1. 特殊机制实现（增援、适应性等）
2. 敌人"AI智能"提升
3. 可视化调试工具
4. 性能优化

## 平衡性考虑

### 金币平衡
- 每波起始金币递增
- 金币/秒根据波次调整
- 物品成本与效果匹配

### 时间平衡
- CD时间确保合理的单位密度
- 加速效果不能过于强大
- 给玩家足够的反应时间

### 空间平衡
- 大型物品占用更多格子但效果更强
- 鼓励多样化配置
- 避免单一策略过强

## 扩展性设计
