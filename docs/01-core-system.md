# 游戏核心系统设计文档

## 概述

大扎巴的核心系统管理整个游戏的流程控制、状态管理和波次进程。游戏采用双阶段循环机制，为玩家提供策略规划和战斗观战的完整体验。

## 游戏状态管理

### 游戏状态 (gameState)
- **playing**: 正常游戏进行中
- **paused**: 游戏暂停
- **gameover**: 游戏结束

### 游戏阶段 (gamePhase)
- **preparation**: 备战阶段 - 玩家购买物品、调整阵容
- **battle**: 战斗阶段 - 自动战斗进行中

## 双阶段游戏循环

### 备战阶段 (Preparation Phase)
**持续时间**: 无限制，玩家主动触发进入战斗

**主要功能**:
```javascript
// 备战阶段显示内容
- 敌人介绍区域 (enemy-intro)
- 商店系统 (shop-section)
- 背包管理 (backpack-section)
- 战斗区物品拖拽
```

**玩家操作**:
- 从商店购买物品
- 调整战斗区阵容
- 管理背包存储
- 查看即将到来的敌军信息
- 点击"开始战斗"进入战斗阶段

### 战斗阶段 (Battle Phase)
**持续时间**: 直到一方基地被摧毁

**主要功能**:
```javascript
// 战斗阶段显示内容  
- 战场Canvas画布 (battlefield)
- 隐藏商店和背包
- 单位自动生成和战斗
- AI敌军生产
```

**自动进行**:
- 根据战斗区物品自动生产己方单位
- AI控制敌方单位生产
- 单位自动寻敌、移动、攻击
- 基地血量计算和判定

## 波次系统 (Wave System)

### 总体设计
- **总波数**: 10波
- **胜利条件**: 通过所有10波挑战
- **失败处理**: 基地被摧毁后直接进入下一波（不游戏结束）

### 难度递增机制

**基地血量成长**:
```javascript
// 基地血量公式
baseHealth = 50 + 50 × waveNumber
// 第1波: 100血, 第10波: 550血
```

**敌方AI增强**:
```javascript
// 敌方初始金币
enemyGold += wave × 50
// 保证敌方有足够资源生产单位
```

**波次奖励**:
```javascript
// 每波开始获得金币
waveGold = 5 × waveNumber
// 第1波: 5金币, 第10波: 50金币
```

## AI系统 (Enemy AI)

### AI配置
```javascript
enemyAI = {
    spawnTimer: 0,           // 生产计时器
    spawnInterval: varies    // 生产间隔(基于波次)
}
```

### AI行为逻辑

**单位生产**:
1. 根据波次配置选择兵种类型
2. 检查金币是否充足
3. 控制单位数量上限(最多30个)
4. 按照时间间隔持续生产

**智能限制**:
- 敌方单位数量限制，避免性能问题
- 金币不足时停止生产
- 根据波次调整生产策略

### 内存管理

```javascript
// 每波结束时清理资源
this.renderer.clearAllEffects();
this.inventorySystem.clearDOMCache();
if (window.gc) window.gc(); // 强制垃圾回收
```

### 单位数量控制
```javascript
// 性能保护：限制单位总数
const maxPlayerUnits = 50;

```

### 计算优化
- 排斥力计算：每3帧计算一次
- 目标搜索：每10帧重新搜索一次
- UI更新：每10帧更新一次

## 事件系统

### 核心事件绑定
```javascript
// 开始战斗按钮
document.getElementById('start-battle-btn')
    .addEventListener('click', () => this.startBattle());

// 游戏结束重启
document.getElementById('restart-game')
    .addEventListener('click', () => window.location.reload());

// 窗口大小调整
window.addEventListener('resize', () => this.renderer.resize());
```

### 生命周期管理
1. **init()**: 初始化游戏状态、创建基地、绑定事件
2. **gameLoop()**: 主循环，每帧执行一次
3. **updateUnits()**: 更新所有单位状态
4. **updateAI()**: 更新AI行为
5. **checkGameOver()**: 检查胜负条件
6. **render()**: 渲染所有游戏元素

## 胜负判定

### 战斗胜利
- 敌方基地血量归零
- 进入下一波备战阶段

### 战斗失败
- 玩家基地血量归零
- **不游戏结束**，直接进入下一波
- 显示失败提示但继续游戏

### 游戏胜利
- 成功通过第10波
- 显示胜利界面
- 提供重新开始选项

### 游戏机制特色
- **宽松失败机制**: 单波失败不影响整体进程
- **渐进式挑战**: 每波有不同特色和难度
- **策略深度**: 需要针对不同波次调整战术
