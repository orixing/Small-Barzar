# 背包商店系统设计文档

## 概述

背包商店系统是游戏的核心经济和物品管理模块，负责处理商店购买、物品存储、拖拽交互和物品升级等功能。该系统为玩家提供了深度的策略性物品组合玩法。

## 系统架构

### 核心数据结构

```javascript
class InventorySystem {
    inventory: Array[6],        // 战斗区 6×1 格子
    backpack: Array[6],         // 背包 6×1 格子  
    shopItems: Array[3],        // 商店 3个物品槽
    refreshCost: 1,             // 刷新费用
    backpackOpen: false,        // 背包开关状态
    refreshesUsedThisWave: 0    // 本波已用刷新次数
}
```

### 物品分类体系

#### 近战单位 (melee)
**小型单位 (1×1)**:
- **战士** ⚔️ - 基础均衡单位
- **忍者** 🥷 - 购买近战时自我强化 
- **角斗士** 🪓 - 击杀敌人永久攻击力+5
- **野蛮人** 👹 - 相邻近战物品攻击力+10
- **武士** 🥋 - 相邻单位召唤时增强自身

**中型单位 (2×1)**:
- **巨人** 💪 - 非近战伤害减免40%
- **骑兵** 🐎 - 高机动性单位

**大型单位 (3×1)**:
- **民兵团** 👪 - 每次战斗召唤数量递增
- **泰坦** 🗿 - 800护盾值，超强单体

#### 其他兵种
- **战弓** 🏹 (ranged) - 远程攻击单位
- **盾牌** 🛡️ (tank) - 高防御坦克单位
- **法杖** 🪄 (mage) - 被动加成，不召唤单位

#### 魔法物品 (mage)
- **学徒** 🧝‍♂️ - 每有魔法物品攻击力+
- **水元素** 🌊 - 中型魔法物品
- **魔偶** 🤖 - 相邻魔法物品时攻击力大幅提升
- **奥数魔像** 🧞‍♂️ - 相邻魔法物品时获得护盾
- **大法师** 🧙‍♂️ - 1小1中1大组合时全场加速
- **药剂师** ⚗️ - 加速相邻魔法物品
- **女巫** 🧙🏿‍♀️ - 多目标攻击(紫2/橙3/红4)
- **魔法巨龙** 🐉 - 范围攻击，递减伤害

#### 支援物品
- **战旗** 🚩 - 加速其他近战单位召唤
- **宝剑** 🗡️ - 为相邻单位提供攻击加成
- **徽章** 🏅 - 持续减少左侧单位冷却
- **实验室** 🔬 - 每个魔法物品给全场+8攻击力
- **魔力水晶** 💎 - 主动充能其他物品
- **炼金工房** 🏭 - 产生金币和出售加成
- **魔法阵** ✡️ - 所有魔法物品+1攻击目标

## 商店机制

### 商店刷新逻辑

**首次生成特殊机制**:
```javascript
// 50%概率生成近战组合
combo1 = [忍者(绿), 骑兵(绿), 法杖(绿)]

// 50%概率生成魔法组合  
combo2 = [学徒(绿), 战士(绿), 水元素(绿)]
```

**后续生成权重系统**:
```javascript
sizeWeights = {
    1: 60%,  // 1×1物品更常见
    2: 30%,  // 2×1物品
    3: 10%   // 3×1物品较稀有
}
```

### 品质生成规则

1. **波次限制**: 每个品质有最低出现波次
2. **已有物品检测**: 优先生成玩家已拥有的物品品质
3. **升级机制**: 购买相同物品会自动升级品质

### 价格计算公式

```javascript
// 基础价格计算
basePrice = itemTemplate.baseValue
finalPrice = basePrice × (2 ^ qualityLevel)

// 出售价格（50%回收）
sellPrice = finalPrice / 2
```

## 背包系统

### 存储机制
- **战斗区**: 6个格子，物品自动生产单位
- **背包**: 6个格子，仅存储不生产
- **大小适应**: 支持1×1、2×1、3×1不同尺寸物品

### 拖拽系统

**支持的拖拽操作**:
1. 商店→战斗区: 购买并放置
2. 商店→背包: 购买并存储
3. 战斗区↔背包: 双向移动
4. 战斗区内部: 重新排列
5. 物品→商店区域: 出售操作

**智能推挤系统**:
```javascript
// 拖拽时自动推挤其他物品为新物品腾出空间
pushItemsAway(targetIndex, itemSize) {
    // 计算是否能推挤
    // 执行推挤操作
    // 更新布局
}
```

## 物品升级系统

### 升级条件
- 拥有相同ID和品质的物品
- 购买相同物品时自动合并升级
- 品质上限为5级（神话）

### 升级效果
- **属性提升**: 每级25%基础属性加成
- **特殊技能增强**: 部分物品升级会增强技能效果
- **视觉反馈**: 升级时显示特效和颜色变化

## 特殊技能系统

### 被动技能类型

**相邻效果**:
- **野蛮人**: 相邻近战+10攻击力
- **魔偶**: 相邻魔法物品时攻击力提升
- **徽章**: 左侧单位冷却减少

**全局效果**:
- **法杖**: 全场魔法物品+5攻击力
- **实验室**: 每个魔法物品给全场+8攻击力
- **魔法阵**: 魔法物品+1攻击目标

**累积效果**:
- **角斗士**: 击杀敌人永久+5攻击力
- **武士**: 相邻单位召唤时自身+2攻击力
- **民兵团**: 每次战斗召唤数量+1

### 主动技能
- **战旗**: 触发全场近战单位加速
- **药剂师**: 召唤时加速相邻魔法物品
- **炼金工房**: 产生金币
- **魔力水晶**: 充能其他物品

## 经济系统

### 金币来源
1. **波次奖励**: 每波开始获得 `5 × 波数` 金币
2. **炼金工房**: 主动产生金币
3. **出售物品**: 50%价格回收

### 金币消耗
1. **购买物品**: 根据品质等级定价
2. **商店刷新**: 免费但每波限1次

### 刷新机制
- **每波限制**: 每波最多免费刷新1次
- **重置时机**: 新波开始时重置刷新次数
- **自动刷新**: 新波开始免费刷新商店

## 战斗阶段行为

### 自动生产机制
```javascript
// 战斗阶段每帧检查
autoProduceUnits() {
    for (item in battleItems) {
        if (item.isReady && item.unitCount > 0) {
            produceUnit(item);
            resetCooldown(item);
        }
    }
}
```

### 冷却时间系统
- **基础冷却**: 每个物品有基础冷却时间
- **品质影响**: 部分物品品质影响冷却
- **被动加成**: 徽章等物品可减少冷却时间
- **战斗重置**: 每波结束所有进度条归零

## 用户交互优化

### 移动端适配
- **大尺寸触摸区域**: 适合手指操作
- **拖拽反馈**: 视觉高亮和震动反馈
- **防误触**: 合理的触摸判定区域

### 视觉反馈系统
- **品质颜色**: 物品边框显示品质颜色
- **攻击力标签**: 显示单位攻击力数值
- **冷却进度条**: 实时显示生产进度
- **升级特效**: 升级时的动画效果

### 信息展示
- **物品悬停**: 显示详细属性信息
- **技能描述**: 清晰的技能说明文本
- **价格指示**: 根据购买力显示价格颜色

## 性能优化

### 内存管理
- **DOM缓存清理**: 每波结束清理DOM缓存
- **事件解绑**: 动态创建的元素及时解绑事件
- **对象池**: 重用物品对象减少GC压力

### 计算优化
- **技能计算缓存**: 避免重复计算相同技能效果
- **批量更新**: 集中更新UI避免频繁重绘
- **延迟加载**: emoji等资源预加载机制
